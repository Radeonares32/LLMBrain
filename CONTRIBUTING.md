# Contributing to LLM Brain

First off, thank you for considering contributing to LLM Brain! It's people like you that make LLM Brain such a great tool for source-grounded engineering memory compilation.

## Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our [Issues](https://github.com/LLMBrain/issues) first to see if someone else has already created an issue. If not, go ahead and make one!

## Fork & create a branch

If this is something you think you can fix, then fork LLM Brain and create a branch with a descriptive name.

## Get the development environment running

1. Clone your fork.
2. Install dependencies.
3. Run the tests to make sure everything is working.

## Implement your fix or feature

At this point, you're ready to make your changes! Feel free to ask for help; everyone is a beginner at first.

## Make a Pull Request

At this point, you should switch back to your master branch and make sure it's up to date with LLM Brain's main branch.
Then create a Pull Request with a clear list of what you've done. When you send a pull request, we will love you forever if you include tests. We can always help with this.

## Code Review

Once you've submitted your pull request, the maintainers will review your code. We may ask you to make some changes before your pull request is merged.

## Community

Join our community to discuss the project and get help with contributing.
